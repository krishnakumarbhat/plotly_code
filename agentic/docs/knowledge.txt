Work Agent Notes

This is a tiny local RAG demo project called Work.
It uses a local GGUF model with llama.cpp and ChromaDB for retrieval.

RAG best practices in this project:
- Keep chunks small (about 300-500 characters) for focused retrieval.
- Persist vector DB so indexing runs once.
- Always pass retrieved context into the final answer prompt.
- Use local files to keep data private.

Troubleshooting:
- If model loading fails, reduce context size (n_ctx) and increase RAM.
- If response is slow, reduce max_tokens or use fewer retrieved chunks.
